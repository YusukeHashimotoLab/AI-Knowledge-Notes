# 修正案

## 1. テーマ統一とダークモード対応
- トップページ (`private/en/index.html:49`) のみ CSS カスタムプロパティとダークモード対応を導入済みですが、他ページでは依然として固定色が埋め込まれています（例: `private/en/research.html:26`, `private/jp/index.html:32`, `endowed/jp/index.html:152`）。共通スタイルシートで変数を定義し、全ページが同じテーマと `prefers-color-scheme` に追従するよう移行してください。
- `meta name="theme-color"` は `private/en/index.html:15` や `private/jp/index.html:15` などでライトテーマの値に固定されています。ダークモードにも対応した `<meta name="theme-color" media="(prefers-color-scheme: dark)" ...>` を追加し、ブラウザの UI カラーが背景と乖離しないよう整備してください。

## 2. サイト間ナビゲーションと導線改善
- 研究トップ (`private/en/index.html:571`) のナビゲーションは同一セクションのみを指し、寄附講座サイト (`endowed/jp/index.html:706`) にはそもそもグローバルナビが存在しません。共通ナビゲーションコンポーネントを用意し、研究紹介・寄附講座・教育コンテンツ (`knowledge/*/index.html`) を横断できるリンクを追加してください。
- フッターの連絡先 (`private/en/index.html:706`, `private/jp/index.html:501`) にも外部サイトや関連プロジェクトへの導線を加えると、二度クリックしないと情報に辿り着けない課題を緩和できます。

## 3. ヒーローセクションとテキスト構造
- 英語トップのヒーローコピーは `<br>` を多用しています (`private/en/index.html:588-592`)。文節ごとにコンテナーを分け、`max-width` と `clamp()` を利用したフォントサイズ制御に置き換えることで、読み上げや狭幅表示でも自然な段落になります。
- 同セクション内ボタン群 (`private/en/index.html:595`) には `btn-primary` / `btn-secondary` が配置されていますが、縦積み・横並び切り替えのためのユーティリティクラスが未整備です。`.stack` や `.cluster` などの共通レイアウトヘルパーを導入し、他ページでも繰り返し利用できるようにしてください。

## 4. ニュース・イベント管理フロー
- 英語ニュース (`private/en/news.html:454-519`) と日本語ニュース (`private/jp/news.ja.html:436-493`) を HTML で別管理しているため、将来同時更新が難航します。JSON/YAML から言語別にビルドし、翻訳待ちは「Translation in progress」等のステータスを自動付与する仕組みを導入してください。
- トップページの Selected Publications & Presentations セクション (`private/en/index.html:694-729`) は手動で Upcoming バッジを切り替えています。開催日や公開日を基準に表示順・バッジを自動更新できるビルド処理を追加し、イベント終了後にアーカイブへ移すフローを整備してください。
- 寄附講座ページのニュース (`endowed/jp/index.html:541-672`) は未来日付と公開済みが混在したまま表示されています。公開ステータスのフラグ管理と、確定済みのみ配信するレビュープロセスを導入してください。

## 5. コード共通化と保守性
- 大量のインライン CSS が各ページに重複しています（例: `private/en/index.html:32-520`, `private/en/news.html:23-388`, `private/jp/index.html:32-520`）。`assets/css/base.css` などに共通レイヤーを切り出し、ページ固有のスタイルだけを局所化することで、パフォーマンスと保守性を改善できます。
- Markdown と HTML が重複するファイル（`knowledge/en/index.md` と `knowledge/en/index.html` など）が多数存在し、更新漏れの温床になっています。静的サイトジェネレーターやテンプレートエンジンを導入し、単一ソースから各言語・各ページを生成するパイプラインへ移行してください。

## 6. 多言語コンテンツ整合性と翻訳フロー
- 日本語トップのみ Impact セクションを備えています（`private/jp/index.html:585`）が、英語トップ（`private/en/index.html`）には対応する実績紹介がありません。日英でセクション構成を揃え、翻訳待ちのブロックには「Coming soon」などのプレースホルダーを置くと閲覧者に不整合を感じさせません。
- 研究紹介・ニュース・リンク集など言語別 HTML が完全に別コピーで維持されています（例: `private/en/news.html` と `private/jp/news.ja.html`）。Front Matter 付きの共通 Markdown や YAML を源泉にし、翻訳ステータス（`draft` / `needs-review` 等）を管理することで、更新の抜け漏れと片側のみの公開を防げます。
- 教育コンテンツ導線が各言語トップから欠落しています。`private/en/index.html:595` や `private/jp/index.html:555` の CTA に `knowledge/en/index.html` / `knowledge/jp/index.html` へのリンクを追加し、研究紹介から教育リソースへ自然に遷移できるようにしてください。

## 7. アクセシビリティとユーザー体験の改善
- 日本語トップではページ読み込み時に強制スクロールを実行しています（`private/jp/index.html:628`）。アンカー付き URL を共有すると閲覧者が先頭へ戻されるため、スクリプトを削除し `history.scrollRestoration` のデフォルト挙動に任せてください。
- `.btn` コンポーネントは hover のみカスタムスタイルがあり、キーボード操作時にフォーカスが視認できません（`private/en/index.html:258`、`endowed/jp/index.html:190` など）。`:focus-visible` で輪郭とコントラストの高い影を追加し、WCAG 2.1 の要件を満たしてください。
- アイコンのみのコンテンツ（例: `private/en/index.html:646` のカードアイコン、`endowed/jp/index.html:596` の絵文字）にはスクリーンリーダー向けに `aria-hidden="true"` と説明テキストを併記し、情報が視覚以外でも伝わるようにしてください。
