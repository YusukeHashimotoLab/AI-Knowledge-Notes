# 修正案

## 1. コンテンツの信頼性と更新フロー
- `private/en/index.html:694` や `private/en/news.html:454` では Upcoming バッジ付きのイベント情報を手動で管理しています。開催後に自動でアーカイブされ、掲載前レビューを通過した情報のみが公開されるよう、単一データソースとリリースチェックリストを整備してください。日本語版のニュース (`private/jp/news.ja.html:436`) も同じソースを参照できると更新漏れを防げます。
- `endowed/jp/index.html:541` 以降のニュースとプレスリリースにも未来日付が混在しているため、公開予定の内容はドラフト扱いにし、確定済み情報のみが表示される CMS 運用（公開ステータスのフラグや自動非表示機能など）を検討してください。

## 2. 情報設計・多言語運用
- `private/en/news.html:454-519` と `private/jp/news.ja.html:436-493` は現在別々の HTML を同期しているため、JSON/YAML などの単一データソースから多言語ページをビルドし、翻訳中の項目には状態ラベルを表示するワークフローを導入してください。
- グローバルナビがサイト単位で分断されているので、`private/en/index.html:571` のナビゲーションや `endowed/jp/index.html:706` のヘッダー構造を共通コンポーネント化し、`knowledge/*/index.html` からも研究紹介・寄附講座サイトへ戻れる導線を追加してください。

## 3. レイアウトとレスポンシブ対応
- `private/en/index.html:588` のヒーローテキストは `<br>` で強制改行しているため、画面幅によって不自然な折返しが発生します。`max-width` 指定や `clamp()` を用いたフォントサイズ調整で、マークアップを素直に保ちながら柔軟にレイアウトできるようにしてください。
- `private/en/index.html:700` 周辺ではボタンを縦積みにするため `btn-full` を使用していますが、カードごとに margin を個別指定しています。共通ユーティリティ（例: `.stack > * + * { margin-top: ... }`）に置き換えることで、長期的な保守性と一貫性を高められます。

## 4. アクセシビリティとユーザー体験
- `endowed/en/index.html:992` や `private/en/publications.html:438` など、複数ページに `window.scrollTo(0, 0);` が残っているため、ブラウザの履歴戻りでスクロール位置がリセットされます。履歴操作時には処理を抑制するか、スクリプト自体を削除してください。
- `private/jp/contact.ja.html:523` など日本語ページはダークモード対応が未整備です。`private/en/index.html:49` のように CSS カスタムプロパティと `@media (prefers-color-scheme: dark)` を導入し、全ページで配色切り替えを統一してください。

## 5. パフォーマンスと保守性
- `private/en/index.html:32-412` や `private/en/news.html:32-332` など大規模なスタイル宣言が依然として各 HTML に埋め込まれています。共通 CSS を `assets/css/base.css` として切り出し、`<link rel="stylesheet">` 参照に切り替えることで、読み込みサイズと保守コストを削減できます。
- コードと文章が同居する `.md` / `.html` の重複 (`knowledge/en/index.html` と `knowledge/en/index.md` など) は更新抜けの原因になります。静的サイトジェネレーターやテンプレートエンジンを導入し、単一ソースから HTML を生成するパイプラインに移行してください。
